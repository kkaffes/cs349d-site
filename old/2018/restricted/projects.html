CS349D Project Ideas

The following is a list of potential topics and ideas for your CS349d project. These are starting points that will require further development. Please reach out to the instructors for futher information.

Machine Learning

  • Design an infrastructure for mobile-edge neural network computations. Offload to edge servers or datacenter servers as needed for performance/power or based on data availability.
  • In a cloud ML system as new data arrive, how do we decide when to re-train or incrementally train a ML model? How does the cost vs. benefit relate to the model type? What infrastructure is needed to support this?
  • ML Serving: Design a system to automatically generate fast model serving code from a data science pipeline in libraries like Pandas and SciKit-Learn, using technology such as Weld.
  • ML and Hardware: build a parameter server for distributed ML training that exploits 3D crosspoint memory to increase capacity without introducing performance problems.

Serverless Computing

  • Serveless for high throughput serving systems: Serverless computing is quite popular with low volume serving applications. For high volume serving application (e.g., websearch), people still use traditional server designs. Can we design a serverless system that can execute efficiently the same code both at low and high volume scenarios?
  • Serverless for accelerators: Does it make sense to extend the serverless platforms for FPGAs, and other accelerators? If yes, how does the system look like?
  • Build a serverless platform based on slack compute, memory, and storage resources.
  • The economics of serverless: How should one charge for serverless? Alternatively, given the economics and the usage characteristics, how should one set the thresholds for constraints (e.g., up to 500 MB disk)?

Cloud Management

  • ML for cloud management: Several researchers are suggesting the use of reinforcement learning (RL) in cloud management. Select on cloud management task (e.g., resource allocation, scheduling, etc) and build RL solutions to it.
  • Several companies will always need to be partially on-premise or will need to operate using >1 cloud operator. What do you need to change or optimize in cloud frameworks (orchestration, analytics, serving) to deal with multi-cloud case? E.g., how do you optimize around slow inter-cloud networks or reconcile the different security models?

Security and Privacy

  • Multi-party computation as a service.
  • Edge computation as a service: design a platform for running functions on the edge (IOT) while isolating them from each other the way a cloud does.
  • Privacy: Implement DIY applications using serverless computing that users can use for a common computing task, such as chat or email.
  • Analytics and Privacy: Design a managed storage & analytics stack that automatically respects data privacy regulations such as GDPR (e.g. keeping data for each country inside it).

Storage

  • Storage and Economics: Design a system that automatically compresses data or moves it to faster/slower tiers of storage to minimize cloud cost subject to meeting a certain workload SLA.
